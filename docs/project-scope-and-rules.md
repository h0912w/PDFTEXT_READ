# Project Scope and Rules

## 프로젝트 핵심 정의
이 프로젝트는 일렬 배치된 텍스트가 포함된 PDF를 입력받아, 텍스트 레이어 추출과 OCR/비전 분석을 결합해 사람이 읽는 순서로 텍스트를 복원하고, 검수 가능한 CSV/XLSX를 생성하는 시스템이다.

## 목표
- 디지털 PDF에서는 텍스트 레이어의 문자 정확도를 최대한 활용한다.
- 스캔본 PDF에서는 OCR/비전 기반으로 실무 사용 가능한 결과를 만든다.
- 하이브리드 PDF에서도 페이지/영역별 우선순위 적용으로 결과를 복원한다.
- 검수용 XLSX에서 원본 이미지와 결과 텍스트를 즉시 대조할 수 있게 한다.
- 변경 후에도 샘플 PDF 기반 QA로 품질을 유지한다.

## 비목표
- 범용 OCR 플랫폼 구축
- 모든 PDF 배치 유형의 완전 일반화
- 복잡한 다단/표 문서 전체 복원
- 사람이 전혀 검수할 필요 없는 시스템 보장
- 문서 편집 기능 제공

## 항상 지켜야 할 규칙
- 정확도와 순서 복원 품질을 우선한다.
- 디지털/스캔본/하이브리드 분기를 먼저 결정한 뒤 이후 단계를 수행한다.
- 일부 텍스트가 불확실해도 실행 단계에서는 그 텍스트만 스킵하고 전체 실행을 계속한다.
- 스킵 항목은 반드시 로그와 산출물에 남긴다.
- QA 단계에서는 스킵을 허용하지 않는다.
- QA는 코드 변경뿐 아니라 문서 변경 후에도 필수다.
- QA 샘플과 정답 데이터는 코드와 분리된 독립 디렉터리에서 관리한다.

## 우선순위 충돌 시 판단 기준
1. 정답과의 일치성
2. 읽기 순서 자연성
3. 출력 계약 유지
4. 검수 편의성
5. 실행 지속성
6. 코드 구조 개선

## 에이전트/스크립트 역할 원칙
- 판단, 해석, 승인/반려는 에이전트가 담당한다.
- 반복 가능한 추출, 계산, 저장, 비교는 스크립트가 담당한다.
- 스크립트로 확정 가능한 규칙을 에이전트 추론으로 바꾸지 않는다.
