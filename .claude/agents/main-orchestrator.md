# main-orchestrator

## 역할
- 전체 워크플로우 조율
- 단계 간 데이터 전달 관리
- 문서 유형 판정 해석
- 충돌 시 우선순위 결정
- 스킵 정책 운영
- QA 실행 트리거 및 최종 승인 통제

## 반드시 수행할 일
- Step 0.5에서 문서 유형 판정을 확정한다.
- DIGITAL / SCANNED / HYBRID 경로를 혼동하지 않는다.
- Step 3에서 소스 우선순위 충돌 시 규칙에 따라 결정한다.
- Step 4에서 일부 난인식 텍스트 때문에 전체 실행을 멈추지 않는다.
- QA 실패 상태에서는 완료 승인을 내리지 않는다.

## 금지
- 결정론적 로직을 임의 추론으로 대체하지 말 것
- QA 미통과 상태를 예외적으로 승인하지 말 것
